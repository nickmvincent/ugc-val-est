Answered posted on: 2012-02-11 22:03:21+00:00 If you by "the sorted array has the same elements" mean that the sorted array has the some number of occurrences for each element, then those two tests are sufficient. (An array such as { 1, 1, 2 } should not be a valid result for the input {2, 1, 2}.) Another way of expressing it is that the result should be a permutation of the input.

A third requirement which may or may not interest you is whether or not the algorithm is ***stable. In that case you should add a test that makes sure that any two equal elements in the input have the same relative order in the output.
