Answered posted on: 2011-12-04 23:11:45+00:00 There is the websockets API, but it is not supported on all browsers. The latest draft is supported by Chrome 14, Firefox 7 and Internet Explorer 10. If this is for development only, and you use on of those browsers, that shouldn't be a problem.

This API will allow the server to contact the client as well as the other way around, so no polling needed anymore. See also the ***Wikipedia page for an introduction.
