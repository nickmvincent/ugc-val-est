Python's built-in json module loads and dumps entire objects, so you can't do it directly with that. There are event-driven ("SAX style") JSON parsers available, for example yajl, which has Python bindings.

Personally I try not to use 3rd-party dependencies unless I have to, so I'd probably go with manually printing the beginning/end/keys and using the built-in JSON module. Or just keep the whole thing in memory. How much memory does it actually take? Remember to only optimize once you've profiled and found it to be a problem. :-)
