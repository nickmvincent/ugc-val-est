As you can see, every other column is of the type or could be or even etc.

As it's defined by the beans with the PropertyValueFactory:

emailCol.setCellValueFactory(
        new PropertyValueFactory<Person, String>("email"));

However, the first column you don't just one pass an information of the person as the class Person doesn't have any index or other number. So when you create the number column as:

TableColumn<Person, Person> numberCol = new TableColumn<Person, Person>("#");

You are creating a "meta" column and by that you are using persons to create the elements of such column (as you can see you get the index of the person to fill it). In fact the column have Persons, but its purpose is to have a column which will complete information that aren't exactly from the class Person but is relevant to be in the table.
